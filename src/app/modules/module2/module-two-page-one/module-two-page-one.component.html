Buttons Galore!

Buttons are something that everyone knows about in real life: a small thing that you press and then something new happens, like a doorbell. Just like in real life, buttons on a webpage work the same way. You can also set the behavior that your button will exhibit after a user interacts with the button, which lets you set your button to do something like change the current web page or update a feature of the page. Your first question after learning about buttons might be ‘how do I actually make one?’ and to answer that question, you would use the button tag denoted by:

<button> </button>

You can also set the text of a button to whatever you want by placing that text between the two button tags. Once you do this, you can then modify the button to set its behavior. There are many different behaviors that you can set, but just to start we’ll go with the most basic type of behavior: clicking! To set this, in the opening button tag you would add ‘(click)= ” ” ’ where everything within the double quotes is the actions that your program will take after clicking on the button.

This is especially useful when trying to connect your html to more advanced languages such as TypeScript or JavaScript, which allows your html button to call a function in one of those languages. We know you may not have any experience with another coding language, and that's perfectly fine for now! The only thing that you need to know is that you can make something called a function, and a function takes in input through something called a parameter.

A function is just a chunk of code that can be repeated by just referring to one thing. Think about how turning on your computer is a very complicated process, but all you need to know how to do is press the power button. If that were a function, you could say that its name would be turnOn and every time you press the button you call the function turnOn that causes your computer to do everything it needs to do to actually turn on. A parameter is a value that you have to give a function, and that function can then use it to complete its task.

The microwave doesn’t have the ability to do anything to your food until you put it inside, which is exactly how a parameter works with a function. To give you an example of this would practically work, you call a function by just typing its name, and then immediately follow it with a set of parentheses. Within the parenthesis you can include a parameter if you need one. This would look something like this:

useMicrowave(cheesePizza)

Or if there was no parameter:

turnOn()

For example of what this would look like within a button, if you have a TypeScript function called ‘changeColor()’ and it accepts one parameter called ‘hexColor’ you could use a button to allow the user to call the changeColor function with a set value.

In this case, we are going to use CSS :active to emulate a similar concept without the use of Javascript or Typescript.

It is also important to know that you can still apply all the usual things to buttons like stylings. For example, if you wanted to shift the button to right you could set the style attribute to something like margin-left: 5%’ and that would be perfectly valid here!
