What is CSS?

CSS stands for Cascading Style Sheets, and is the code that allows us to apply styles to an HTML element. Remember all those times you wrote styles=”” in the previous modules? That was actually CSS! There are two main ways to use CSS that we’ll be covering in this course: inline CSS and external CSS. Inline CSS is the method that you’ve been using so far, where you write the CSS within the same line as your HTML hence the name inline CSS. External CSS relies on a separate CSS file that contains all the different styles that you would like to apply to a given HTML element. The neat thing about external CSS is that you can define a whole bunch of changes to the style under one name, and then just call that name in the HTML file to have every change applied.

At this point, you’ve gotten plenty of experience with inline CSS, so we’re going to focus on external CSS. How do you actually write CSS in this way though, you may be asking. It is actually quite simple, you just need to follow this convention:

selector {

property: value;

}

The selector is the name associated with the grouping of properties that are being set. The property here refers to something like the background color, or font size, it's the name of the thing we want to change. The value here is the actual data that is being used, if it were a color it might be ‘blue’ or if it were a number it could be ‘5%’. If there were multiple things being changed you would simply repeat the ‘property: value;’ pattern on a new line.

You can also set multiple selectors to have the same property and value by separating them with a comma followed by a new line. For example, you could set multiple selectors to have a property value of red with the following code:

Thing1,

Thing2,

Thing3 {

color: red;

}

You can set an HTML element to have these stylings applied by using the ‘class’ attribute and typing each selector’s name between the quotations with each one separated by a space. One important thing to note about CSS is that different styles cascade. What this means is that if you have two different selectors that modify the same value, the one that is lower in your CSS file will be the one that is selected. For example, if your CSS file looked like this:

foo {

color: red;

}

bar {

color: blue;

}

And then you applied this to a div with the following line:

<div class=”bar foo”>What’s the right color? </div>

The color of the text would be blue, not because ‘bar’ came first, but because it is the lowest selector that modifies the same property. If the stylings were to be reversed as ‘foo bar’ the color of the text would still be blue.

It is also important to know that using inline CSS will override the external CSS used, so you should be careful when trying to mix the two together!
