Merging

Let’s say that in your repository, there are two branches:

- One main branch

- Emily's branch

Emily’s branch is her own personal branch where she has been making all of her changes. The main branch is where all approved code goes after testing is done. The main branch is associated with all ready to go and published changes of the code.

Emily’s code has been tested, and she is ready to combine her branch into the main branch. This can be done by opening a “Pull Request” on the GitHub website.

Navigate to your repository and select “Pull Request” from the menu of options listed. On this tab, click the green button titled, “New pull request”

Select what branch should be merged into the other. In Emily’s case the base branch is “main” and the compare branch is “Emily”.

After the two branches have been selected, click “Create Pull Request”. This will bring you to a page where all commits associated with this merge are visible. If there are any merge conflicts, these can also be resolved here. Once these are all resolved, you can click the “Merge pull request” button that is green in the middle of the page. This will merge Emily’s changes into the main branch.
