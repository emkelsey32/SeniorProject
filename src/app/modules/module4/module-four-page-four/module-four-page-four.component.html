Branches

Branches are an important tool for seamlessly collaborating on the same files in a project. When two people are working on the same branch, and push different changes, this can result in a merge conflict if the changes contradict each other. For example, say that Emily wanted to add a line of code that added two numbers. But Chris wanted to add that same line of code but instead subtract two numbers. Which version is correct?

By working in separate branches, this can be avoided for the time being during development. By working in separate branches, it allows all changes to be pushed to the repository and reviewed by other team members while also not impacting other team members who may also be working on that same file.

To create a branch and switch to it run: git checkout -b [name of branch]
