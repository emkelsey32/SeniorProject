Why Use Git?

Git is a powerful tool that enables users to share code bases with other people. Think of other collaborative services you have used for projects in the past. Google docs for writing or Figma for making online whiteboards are similar services. But GitHub expands this service to code!

GitHub allows for users to create an account and upload files, specifically for coding projects. In essence, GitHub can act as an online file storing service. But the power of GitHub comes from collaboration. With multiple developers working on one page, it is important to keep team members up to date on what version of the code they use and what changes have been made in the past. By pooling together all of the resources into one place, this improves accessibility for all team members.

Additionally, when a change is pushed to a code base, these changes are highlighted and the differences between your local version and the remote version that was pushed to are highlighted. This allows all users to clearly see the changes made and decide whether they should be kept or not.
