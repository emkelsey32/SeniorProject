Commit and Push

After you have added code using the git command, it it time to commit these changes. Commit takes the changes that you staged and adds them to a permanent snapshot of your code files at the time of the commit. This will associate all of these changes with a unique identifier. To commit files run git commit -m [INSERT MESSAGE]. The message you insert should be a brief description of what was changed, no more than a few words.

Now that you have committed your files, it is time to push them to the repository itself. Up until this point, all of your changes have remained just on your local machine. But in order for you to successfully collaborate and publish your changes, the push command needs to be run. Run this command by typing “git push”.

Once this is done, you can return back to the github repository online and see your changes. As you can see, in my repository, a folder titled “Practice” was added, along with a README.md file.
